
## 1. Controlando Ciclos: `break` y `continue`

En Python, los ciclos (`for` y `while`) suelen ejecutarse hasta que terminan su cuenta o su condición es falsa. Sin embargo, a veces necesitamos intervenir.

### El comando `break` (El Interruptor de Emergencia)
El `break` detiene el ciclo **inmediatamente**. Es como si estuvieras en una fila y, de repente, cierran la ventanilla: no importa cuánta gente falte, todo se detiene ahí.

```python
# Ejemplo: Entrada de datos infinita hasta que el usuario decida parar
while True:
    nombre = input("Ingrese su nombre... (escriba 'salir' para terminar): ")

    if nombre == "salir":
        print("Detectada instrucción de salida...")
        break  # Rompe el ciclo 'while' por completo
    
    print("Bienvenido/a: ", nombre)

print("Programa finalizado correctamente.")
```

### El comando `continue` (El Salto de Obstáculos)
A diferencia del `break`, el `continue` no detiene el ciclo, solo **se salta el paso actual** y pasa directamente a la siguiente repetición. Es como si estuvieras revisando una lista de frutas y, si ves una podrida, no dejas de revisar la caja, simplemente la ignoras y pasas a la siguiente.

```python
# Ejemplo: Etiquetado de productos saltando el número 3
for producto in range(6):
    if producto == 3:
        print("--- Saltando producto 3 por control de calidad ---")
        continue # Ignora lo que sigue y vuelve al inicio del for con el número 4

    print("Producto etiquetado número: ", producto)
```

## 2. Ejemplo Integrador: El Escáner de Seguridad

Imaginemos una banda transportadora en una fábrica. Usaremos `break` para emergencias y `continue` para errores menores.

```python
print("="*25)
print("SISTEMA DE ESCÁNER INDUSTRIAL")
print("="*25)

# ESCENARIO 1: Peligro Crítico (Uso de break)
print("\n--- Iniciando revisión de seguridad ---")
for caja in range(1, 11):
    print(f"Analizando caja {caja}...")

    if caja == 5:
        print("¡ALERTA MÁXIMA! Producto peligroso detectado.")
        print("DETENIENDO TODA LA PRODUCCIÓN.")
        break # Detiene el proceso por completo

# ESCENARIO 2: Errores de Proceso (Uso de continue)
print("\n" + "="*25)
print("SISTEMA DE PAGOS BANCARIOS")
print("="*25)

for cliente in range(1, 6):
    print(f"Procesando pago del cliente #{cliente}...")

    if cliente == 3:
        print("AVISO: Fondos insuficientes. Saltando al siguiente cliente.")
        continue # No procesa este pago, pero sigue con el cliente #4

    print(f"¡Pago del cliente #{cliente} realizado con éxito!")
```

Una **Lista** es como un estante con compartimentos numerados. Podemos guardar cosas, sacarlas, cambiarlas de lugar o agregar nuevas.

## Estructuras de Datos: La Tupla (`tuple`)

La **Tupla** es la hermana "seria" de la lista. Su gran diferencia es que es **inmutable**: una vez que la creas, no puedes cambiar sus valores, ni agregar, ni quitar. Se usan para datos que deben permanecer constantes (como las coordenadas GPS o los datos de un producto fijo).

```python
# Ejemplo: Datos de un producto que no deberían cambiar accidentalmente
# (Nombre, Precio, Stock)
producto_fijo = ("Arroz", 1250, 20)

print(f"Producto: {producto_fijo[0]}")

# Esto daría ERROR:
# producto_fijo[1] = 1300  <- Python no permite esto en tuplas
```
